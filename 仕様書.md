# 3Dモデルビューアー 仕様書

## 概要
STEP / STL 形式の3Dモデルファイルを読み込み、画面上に描画・操作できるデスクトップアプリケーション。

---

## 対応環境

| 項目 | 内容 |
|------|------|
| macOS | Apple Silicon (M1/M2/M3/M4/M5) |
| Windows | x86-64 / ARM64 |

---

## 機能一覧

### 1. ファイル読み込み
- メニューバーの「ファイルを開く」ボタンからファイル選択ダイアログを表示
- ドラッグ＆ドロップでファイルを読み込み可能
- 対応形式：`.step` / `.stp` / `.stl`
- 複数モデルを同時に読み込み可能（新しいモデルを開くと既存モデルに追加される）

### 2. 3D描画
- 読み込んだモデルを3D空間に表示
- グリッド（床面）を表示し、モデルの位置感を把握しやすくする
- 照明：環境光 + 指向性ライトで立体感を表現

### 3. モデル操作（マウス）

| 操作 | 動作 |
|------|------|
| 左ボタン ドラッグ | モデルを回転 |
| 右ボタン ドラッグ | モデルを平行移動（パン） |
| スクロール | ズームイン／ズームアウト |

### 4. 平行移動（数値入力）

- ツールバーの「移動」ボタンからダイアログを表示
- X / Y / Z 方向それぞれに移動量を数値入力
- 入力単位：mm、最小設定単位：0.0001mm

### 5. エクスポート

- ツールバーの「エクスポート」ボタンからファイル保存ダイアログを表示
- 現在の平行移動位置を反映した状態で STL ファイルとして出力

### 6. 表示リセット
- ボタン1クリックでカメラ位置・角度を初期状態に戻す

### 7. 原点マーク・座標系
- 座標系：**右手系 Z-up**（X × Y = Z、Z が鉛直上方）
  - X 軸：水平（右方向）
  - Y 軸：奥行き（X × Z から導出、Three.js 内部 -Z 方向）
  - Z 軸：垂直（上方向、Three.js 内部 Y 方向）
- 絶対座標原点（0, 0, 0）に X / Y / Z の3方向を示す矢印マークを常時表示
- X 軸：赤、Y 軸：緑、Z 軸：青
- 各軸の先端にラベル（X / Y / Z）を表示
- 平行移動の入力値はこの座標系に準拠する

### 8. モデル選択と削除
- 3D 描画エリアでモデルをクリックすると選択状態になる（ハイライト表示）
- 空白エリアをクリックすると選択解除
- ツールバーの「削除」ボタン、または `Delete` キーで選択中のモデルを削除
- 同一ファイルから読み込まれた複数メッシュは一括削除

## 技術構成

| 役割 | 技術 |
|------|------|
| デスクトップアプリ化 | Electron |
| 3D描画 | Three.js |
| STEP/STL読み込み | occt-import-js（WebAssembly） |
| 言語 | JavaScript / HTML / CSS |

### 実装上の方針

#### 座標系変換（Z-up → Three.js Y-up）
- Three.js 内部は Y-up だが、アプリケーション全体は Z-up 右手系で統一する
- シーン直下に `zUpRoot`（`THREE.Group`）を配置し `rotation.x = -π/2` を設定
- モデル・グリッド・原点マーク・ラベルはすべて `zUpRoot` の子として Z-up 座標系でオーサリング
- 変換式：Z-up (x, y, z) → Three.js (x, z, −y)

#### 深度精度（logarithmicDepthBuffer）
- `WebGLRenderer` に `logarithmicDepthBuffer: true` を設定
- カメラ Near = 0.001 mm、Far = 10,000,000 mm（10 km）
- 0.0001 mm 単位の精度を維持しつつ大型モデルにも対応

#### 平行移動（行列変換）
- `modelsGroup.matrixAutoUpdate = false` に設定し、`THREE.Matrix4` で変換を管理
- `applyTranslation()` は `modelsGroup.matrix.makeTranslation(x, y, z)` を呼び出すのみ
- 描画時は行列が GPU に渡されるだけで頂点データは変更しない
- STL エクスポート時のみ `translationMatrix * mesh.matrix` を頂点に適用（bake）してファイル出力

#### WebGL リソースの完全破棄（モデル削除時）
- `geometry.dispose()` でバッファを解放
- `material` が保持するすべてのテクスチャ（`map`, `specularMap`, `normalMap`, `bumpMap`, `roughnessMap`, `metalnessMap`, `alphaMap`, `aoMap`, `emissiveMap`, `envMap`, `lightMap`, `displacementMap`）を個別に `dispose()` してから `material.dispose()` を呼び出す

# 3Dモデルビューアー 仕様書

## 概要
STEP / STL 形式の3Dモデルファイルを読み込み、画面上で個別に操作・検証できるデスクトップアプリケーション。

---

## 対応環境

| 項目 | 内容 |
|------|------|
| macOS | Apple Silicon (M1/M2/M3/M4/M5) |
| Windows | x86-64 / ARM64 |

---

## 機能一覧

### 1. ファイル読み込み
- **方式**: メニューバーのボタン、またはドラッグ＆ドロップ。
- **挙動**: 既存のモデルを保持したまま、新しいモデルを追加で読み込む（複数モデルの共存）。
- **対応形式**: `.step`, `.stp`, `.stl`

### 2. モデルの選択管理
- **選択操作**: 3D描画エリアでモデルを左クリックで選択。
- **視覚効果**: 選択されたモデルはハイライト表示（外形線の強調または特定色のエミッシブ表示）。
- **解除**: 空白エリアのクリックで選択解除。
- **状態管理**: アプリケーションは常に「現在選択されているオブジェクト」の参照を保持する。

### 3. モデル操作（マウス）

| 操作 | 動作 |
|------|------|
| 左ボタン ドラッグ | モデルを回転 |
| 右ボタン ドラッグ | モデルを平行移動（パン） |
| スクロール | ズームイン／ズームアウト |

### 4. 平行移動（数値入力：選択モデル・相対指令）
- **前提条件**: モデルが選択されている時のみ、ツールバーの「移動」ボタンが有効化される。
- **入力方式**: X / Y / Z 方向の移動量を入力するダイアログを表示。
- **移動ロジック**: **相対値入力**
    - 入力された値を「現在の座標への加算分」として扱う。
    - 入力後、入力欄の数値は `0` にリセットされる。
- **単位**: mm（最小設定単位：0.0001mm）。

### 5. エクスポート
- **挙動**: 現在画面上に存在するすべてのモデルを、それぞれの相対位置を維持した状態で一つの STL ファイルとして出力する。
- **変換の固定**: 各モデルに適用されている行列変換（Matrix）を頂点データに適用（Bake）して出力。

### 6. 表示リセット
- **ボタン操作**: カメラ位置・角度を初期状態に戻す。モデルの座標（移動量）は維持される。

### 7. 原点マーク・座標系
- **座標系**: **右手系 Z-up**（X × Y = Z、Z が鉛直上方）
  - X 軸：赤、Y 軸：緑、Z 軸：青
  - 各軸の先端にラベル（X / Y / Z）を表示
  - 平行移動の入力値はこの座標系に準拠する

### 8. モデルの削除
- **挙動**: 選択中のモデルのみを削除。
- **メモリ管理**: 削除時、関連する WebGL リソース（Geometry, Material）および `occt-import-js` の WASM メモリを明示的に破棄する。

## 技術構成

| 役割 | 技術 |
|------|------|
| デスクトップアプリ化 | Electron |
| 3D描画 | Three.js |
| STEP/STL読み込み | occt-import-js（WebAssembly） |

### 実装上の方針

#### 座標系変換（Z-up → Three.js Y-up）
- シーン直下に `zUpRoot`（`THREE.Group`）を配置し `rotation.x = -Math.PI / 2` を設定。
- モデル・グリッド・原点マーク・ラベルはすべて `zUpRoot` の子として配置。
- 変換式：Z-up (x, y, z) → Three.js (x, z, −y)

#### 深度精度（logarithmicDepthBuffer）
- `WebGLRenderer` に `logarithmicDepthBuffer: true` を設定。
- カメラ Near = 0.001 mm、Far = 10,000,000 mm。
- 0.0001 mm 単位の精度を維持。

#### 行列による相対移動の実装
- 各モデルグループに `matrixAutoUpdate = false` を設定し、`THREE.Matrix4` で変換を管理する。
- 相対移動の適用（ダイアログ入力値はΔ値、適用後0リセット）:
  ```javascript
  const delta = new THREE.Matrix4().makeTranslation(dx, dy, dz);
  selectedGroup.matrix.multiply(delta);
  selectedGroup.matrixWorldNeedsUpdate = true;
  ```
- STL エクスポート時のみ `modelGroup.matrix × mesh.matrix` を頂点に適用（bake）して出力。

#### WASM メモリの明示的解放
- `occt-import-js` が返す属性配列（`m.attributes.position.array` 等）は WASM ヒープへの view。
- `new Float32Array(array)` でコピーを作成してから `Float32BufferAttribute` に渡すことで、`result` オブジェクトが GC されると同時に WASM ビューへの参照が消える。
- `m.index.array` は `Uint32Array.from()` で既にコピー済み。

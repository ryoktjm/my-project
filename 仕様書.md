# 3Dモデルビューアー 仕様書

## 概要
STEP / STL 形式の3Dモデルファイルを読み込み、画面上で個別に操作・検証できるデスクトップアプリケーション。

---

## 対応環境

| 項目 | 内容 |
|------|------|
| macOS | Apple Silicon (M1/M2/M3/M4/M5) |
| Windows | x86-64 / ARM64 |

---

## 機能一覧

### 1. ファイル読み込み
- **方式**: メニューバーのボタン、またはドラッグ＆ドロップ。
- **挙動**: 既存のモデルを保持したまま、新しいモデルを追加で読み込む（複数モデルの共存）。
- **対応形式**: `.step`, `.stp`, `.stl`
- **複数要素ファイルの扱い**: STEP ファイル内に複数のソリッド／ボディが含まれる場合、それぞれを **独立したモデル** としてシーンに追加する。各モデルは個別に選択・移動・削除・エクスポートできる。
  - 命名規則: `ファイル名 [N]`（例: `assembly.step [1]`、`assembly.step [2]`）
  - STL は単一ソリッドのため、常に 1 ファイル = 1 モデルとなる。
- **非同期処理**: パース処理は Web Worker 上で実行し、UI スレッドをブロックしない。

### 2. モデルの選択管理
- **単一選択**: 3D描画エリアまたはアウトライナーでモデルをクリックすると、そのモデルのみが選択される（他の選択は解除）。
- **複数選択**: `Cmd`（macOS）/ `Ctrl`（Windows）を押しながらクリックすると、選択を追加・解除できる。
- **視覚効果**: 選択されたモデルはそれぞれ **OutlinePass** による青い外形線でハイライト表示。
- **全解除**: 3D描画エリアの空白部分をクリックすると、すべての選択が解除される。
- **状態管理**: アプリケーションは「現在選択されているオブジェクトの集合」を保持する。

### 3. モデル操作（マウス）

| 操作 | 動作 |
|------|------|
| 左ボタン ドラッグ | 視点を回転 |
| 右ボタン ドラッグ | 視点を平行移動（パン） |
| スクロール | ズームイン／ズームアウト |

### 4. 平行移動（数値入力：選択モデル・相対指令）
- **前提条件**: 1つ以上のモデルが選択されている時のみ、ツールバーの「移動」ボタンが有効化される。
- **入力方式**: X / Y / Z 方向の移動量を入力するダイアログを表示。
- **移動ロジック**: **相対値入力**
    - 入力された値を「現在の座標への加算分」として扱う。
    - 複数選択時は、選択されているすべてのモデルに同一のΔ値が適用される。
    - 入力後、入力欄の数値は `0` にリセットされる。
- **単位**: mm（最小設定単位：0.0001mm）。

### 5. エクスポート
以下の2形式でエクスポートできる。**対象は選択中のモデルのみ**（未選択のモデルは出力されない）。

#### STL 書き出し
- **挙動**: 選択中のモデルを相対位置を維持した状態で 1 つのバイナリ STL ファイルとして出力。
- **変換の固定**: 各モデルに適用されている行列変換（Matrix）を頂点データに適用（Bake）して出力。

#### GLB 書き出し
- **挙動**: 選択中のモデルを GLB（glTF Binary）形式で出力。
- **保持される情報**: モデルごとの名前・マテリアル色・行列情報（相対位置）・メッシュ階層。
- **座標系**: Z-up ユーザー座標系で出力（`modelsContainer` を直接エクスポート）。

> **前提条件**: 1つ以上のモデルが選択されている場合のみエクスポートボタンが有効化される。

### 6. 表示リセット
- **ボタン操作**: 読み込み済みのすべてのモデルが画面に収まるようカメラ位置・角度・ズームを自動調整する。モデルの座標（移動量）は維持される。

### 7. 原点マーク・座標系
- **座標系**: **右手系 Z-up**（X × Y = Z、Z が鉛直上方）
  - X 軸：赤、Y 軸：緑、Z 軸：青
  - 各軸の先端にラベル（X / Y / Z）を表示
  - 平行移動の入力値はこの座標系に準拠する

### 8. モデルの削除
- **挙動**: 選択中のすべてのモデルを削除。ツールバーボタンまたは `Delete` キーで実行。
- **前提条件**: 1つ以上のモデルが選択されている時のみ「削除」ボタンが有効化される。
- **メモリ管理**: 削除時、関連する WebGL リソース（Geometry, Material）および `occt-import-js` の WASM メモリを明示的に破棄する。

### 9. 設定機能（外観）
- **設定項目**: アプリケーションの外観（テーマ）を以下の3種類から選択可能。
  - **ライト**: 明るい配色（背景：白系、文字：黒系）
  - **ダーク**: 暗い配色（背景：黒・ダークグレー系、文字：白系）
  - **システム**: OS（macOS/Windows）の設定に同期。
- **反映範囲**:
  - HTML/CSSで構成されるUIパーツ（ツールバー、サイドバー、ダイアログ等）。
  - Three.js 描画エリアの背景色。
  - 3D空間内のグリッドカラー、ヘルパーカラー。
- **永続化**: 選択した設定はアプリケーション終了後も保持され、次回起動時に自動で適用される。

### 10. アウトライナー（モデル一覧）
- **表示位置**: 画面右端のサイドバー。
- **一覧表示**: 読み込まれたモデルのファイル名をリスト表示。読み込み・削除に連動して自動更新。選択状態もリストに反映される。
- **選択**: アイテムをクリックでそのモデルを単一選択。`Cmd`/`Ctrl` を押しながらクリックで複数選択（3D描画エリアと同じ修飾キー規則）。
- **表示/非表示トグル**: 各アイテムの「●/○」ボタンで個別モデルの表示・非表示を切り替えられる。非表示モデルはシーンから隠れるが削除はされない。

## 技術構成

| 役割 | 技術 |
|------|------|
| デスクトップアプリ化 | Electron |
| 3D描画 | Three.js |
| STEP/STL読み込み | occt-import-js（WebAssembly） |
| 言語 | TypeScript |
| ビルド | electron-vite |

### 実装上の方針

#### 座標系変換（Z-up → Three.js Y-up）
- シーン直下に `zUpRoot`（`THREE.Group`）を配置し `rotation.x = -Math.PI / 2` を設定。
- モデル・グリッド・原点マーク・ラベルはすべて `zUpRoot` の子として配置。
- 変換式：Z-up (x, y, z) → Three.js (x, z, −y)

#### 深度精度（logarithmicDepthBuffer）
- `WebGLRenderer` に `logarithmicDepthBuffer: true` を設定。
- カメラ Near = 0.001 mm、Far = 10,000,000 mm。
- 0.0001 mm 単位の精度を維持。

#### Camera-Relative Rendering（RTE）による浮動小数点精度対策
- GPU（WebGL）の float32 は原点から離れた座標でジッターが発生する。原点から 1 m 以上離れると 0.0001 mm 精度を維持できない。
- 毎フレームの描画前に `zUpRoot.position = −cameraTarget` を設定し、カメラの注視点を常にワールド原点付近に保つ。
- カメラ座標は `Spherical` オフセットのみ（cameraTarget を加算しない）で設定。
- CPU 側（JavaScript）では 64-bit float で座標を管理し、GPU には常に小さな差分値のみ渡す。

#### 行列による相対移動の実装
- 各モデルグループに `matrixAutoUpdate = false` を設定し、`THREE.Matrix4` で変換を管理する。
- 相対移動の適用（ダイアログ入力値はΔ値、適用後0リセット）:
  ```javascript
  const delta = new THREE.Matrix4().makeTranslation(dx, dy, dz);
  selectedGroup.matrix.multiply(delta);
  selectedGroup.matrixWorldNeedsUpdate = true;
  ```
- STL エクスポート時のみ `modelGroup.matrix × mesh.matrix` を頂点に適用（bake）して出力。

#### 非同期パース（Web Worker）
- STEP / STL ファイルのパースは `loader.worker.ts`（Web Worker）上で実行。
- UI スレッドのブロックを回避し、パース中も既存モデルの操作・アニメーションを継続できる。
- パース済みの `Float32Array` / `Uint32Array` バッファは `Transferable Objects` としてメインスレッドへゼロコピー転送。
- `occt-import-js` の WASM インスタンスは Worker 内でシングルトンとして保持（再初期化コストを排除）。

#### セレクション表示（OutlinePass）
- `EffectComposer` + `RenderPass` + `OutlinePass` で描画パイプラインを構成。
- 選択中のモデルグループを `OutlinePass.selectedObjects` に渡すことで外形線を描画。
- エミッシブ（発光）方式は廃止。OutlinePass はモデル本来の色・テクスチャを損なわない。
- `onResize()` 時にコンポーザーと OutlinePass の解像度も更新する。

#### WASM メモリの明示的解放
- `occt-import-js` が返す属性配列（`m.attributes.position.array` 等）は WASM ヒープへの view。
- `new Float32Array(array)` でコピーを作成してから `Float32BufferAttribute` に渡すことで、`result` オブジェクトが GC されると同時に WASM ビューへの参照が消える。
- `m.index.array` は `Uint32Array.from()` で既にコピー済み。
